# 법제처 입법예고 자동 동기화 설정 가이드

GitHub Actions가 6시간마다 법제처 정부입법예고 API(data.go.kr)를 서버에서 호출해,
**우리 부서 관련 법령만** 골라 Firestore `legi` 컬렉션에 저장합니다.
앱은 이 컬렉션을 실시간으로 읽으므로 별도 작업 없이 화면에 반영됩니다.

```
GitHub Actions (서버)  ──호출──▶  data.go.kr 정부입법예고 API
       │                              (산업안전보건/위험물안전관리 필터)
       └──쓰기──▶  Firestore(legi)  ──onSnapshot──▶  대시보드 입법예고 섹션
```

대상 법령(스크립트 `RELEVANT_KEYWORDS`에서 수정 가능):
- **산업안전보건** — 산업안전보건법 / 시행령 / 시행규칙 / 기준에 관한 규칙 / 관련 고시
- **위험물안전관리** — 위험물안전관리법 / 시행령 / 시행규칙

---

## 1단계 — data.go.kr 서비스키 발급

1. https://www.data.go.kr 회원가입/로그인
2. **"법제처_정부입법예고"** 검색 → 데이터셋(15058407) 페이지 접속
   - 직접 링크: https://www.data.go.kr/data/15058407/openapi.do
3. 우측 상단 **[활용신청]** 클릭 → 활용목적 입력 후 신청 (오픈 API는 보통 즉시 승인)
4. **마이페이지 → 오픈API → 인증키 발급현황**에서 **일반 인증키**를 확인
   - `Decoding` 키를 사용하세요 (스크립트가 자동 인코딩함)
5. 같은 데이터셋 페이지의 **요청주소(엔드포인트)** 와 **출력결과(응답 필드)** 를 확인
   - 요청주소가 스크립트 기본값과 다르면 → 아래 3단계의 레포 Variable `DATA_API_URL`로 지정

## 2단계 — Firebase 서비스계정 키 발급

1. Firebase 콘솔(hanwha-test-38201) → ⚙️ **프로젝트 설정 → 서비스 계정**
2. **새 비공개 키 생성** → JSON 파일 다운로드
3. 이 JSON **파일 내용 전체**를 다음 단계에서 시크릿으로 등록

## 3단계 — GitHub 시크릿/변수 등록

레포 → **Settings → Secrets and variables → Actions**

**Secrets** 탭 → **New repository secret** 로 2개 등록:

| 이름 | 값 |
|------|-----|
| `DATA_GO_KR_KEY` | data.go.kr 일반 인증키(Decoding) |
| `FIREBASE_SERVICE_ACCOUNT` | 서비스계정 JSON 파일 내용 전체 |

**Variables** 탭 → (요청주소가 기본값과 다를 때만) **New repository variable**:

| 이름 | 값 |
|------|-----|
| `DATA_API_URL` | 데이터셋 명세의 정확한 요청주소 |

## 4단계 — 실행 및 확인

1. 레포 → **Actions → "법제처 입법예고 동기화" → Run workflow** (수동 실행)
2. 로그에서 **"원본 응답 샘플"** 블록 확인
   - 필드명이 스크립트의 후보 목록과 다르면 `scripts/sync-legi.mjs`의 `F` 객체에 실제 필드명을 추가
3. **"관련 법령 … 필터 후 N건"** 로그와 Firestore `legi` 컬렉션에 `gov_…` 문서가 생겼는지 확인
4. 대시보드 입법예고 섹션에 자동 반영됨

이후 6시간마다 자동 실행됩니다(`workflow_dispatch`로 수동 실행도 가능).

---

## 동작 메모

- 각 입법예고는 `gov_<일련번호>` 문서 ID로 **upsert** 됩니다.
- 우리 부서의 **검토 상태(검토전/검토중/완료)** 는 동기화 시 **보존**됩니다(API가 덮어쓰지 않음).
- 첫 동기화 성공 시, 앱이 초기 시드한 데모 입법예고(`l1`~`l5`)는 자동 삭제됩니다.
- 정부 사이트 응답 스키마가 확정되지 않아 필드명을 **방어적으로 다중 매칭**합니다.
  첫 실행 로그로 실제 필드명을 확인해 `F` 매핑을 1줄 보정하면 정확도가 올라갑니다.
